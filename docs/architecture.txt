========================================
АРХИТЕКТУРА ПРИЛОЖЕНИЯ FOKUSMASK
========================================
Версия: 1.0
Дата: Март 2026
Автор: Команда FokusMask

1. ОБЩИЙ ОБЗОР
========================================
FokusMask построен по модульной архитектуре, где каждый компонент
отвечает за свою строго определённую задачу. Это обеспечивает
простоту разработки, тестирования и дальнейшего расширения.

2. ОСНОВНЫЕ КОМПОНЕНТЫ
========================================

2.1. UI Module (Модуль интерфейса)
----------------------------------------
Файл: main.py
Назначение: Отвечает за взаимодействие с пользователем,
отображение элементов управления и обработку событий.
Ключевые элементы:
- Главное окно приложения
- Кнопки-пресеты (3 штуки)
- Кнопка "Стоп"
- Ползунок громкости
Зависимости: Tkinter, Sound Engine

2.2. Sound Engine (Звуковой движок)
----------------------------------------
Файл: sound_engine.py
Назначение: Генерация и воспроизведение звука в реальном времени.
Это сердце приложения.
Что умеет:
- Генерировать белый шум
- Генерировать розовый шум
- Генерировать коричневый шум
- Воспроизводить звук непрерывно
- Останавливать воспроизведение
- Менять громкость
Зависимости: PyAudio, NumPy

2.3. Timer Module (Модуль таймера)
----------------------------------------
Файл: timer_module.py
Назначение: Управление таймерами обратного отсчёта.
Что умеет:
- Запускать таймер на N минут
- Останавливать таймер
- Плавно уменьшать громкость в конце
- Сигнализировать о завершении
Зависимости: threading, Sound Engine

2.4. Settings Module (Модуль настроек)
----------------------------------------
Файл: settings.py
Назначение: Сохранение и загрузка настроек пользователя.
Что сохраняет:
- Последний выбранный пресет
- Уровень громкости
- Состояние таймера
Формат хранения: JSON (settings.json)
Зависимости: json, os

2.5. Audio Assets (Аудиоресурсы)
----------------------------------------
Папка: assets/audio/
Назначение: Хранение звуковых файлов.
Структура:
- white_noise.wav - белый шум
- pink_noise.wav - розовый шум
- brown_noise.wav - коричневый шум
- rain.wav - дождь
- cafe.wav - кофейня
- forest.wav - лес
- sources.txt - информация об источниках

4. ПОТОКИ ДАННЫХ
========================================

Сценарий 1: Запуск приложения
----------------------------------------
1. main.py создаёт окно
2. Загружаются настройки из settings.json
3. Применяется последний пресет и громкость
4. Отображается интерфейс

Сценарий 2: Нажатие на пресет
----------------------------------------
1. Пользователь нажимает кнопку "От соседей"
2. Останавливается текущий звук (если есть)
3. Sound Engine запускает коричневый шум
4. Сохраняется последний выбор в настройки

Сценарий 3: Работа таймера
----------------------------------------
1. Пользователь ставит таймер на 25 минут
2. Timer Module запускает обратный отсчёт
3. По окончании начинается плавное затухание (3 сек)
4. Sound Engine останавливается

7. ПРАВИЛА РАЗРАБОТКИ
========================================
- Модули должны быть независимыми
- Звук работает в отдельном потоке
- Настройки сохраняются при каждом изменении
- Аудиофайлы не больше 10 МБ
